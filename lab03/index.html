Preguntas:

  1. ¿Cuál es la diferencia entre Internet y la World Wide Web?
    Respuesta:

    El internet es una red mundial, la cual usa varios protocolos, mientras que la World Wide Web es el sistema que interconecta muchísimas páginas web a través del mismo internet.

  2. ¿Cuáles son las partes de un URL?
    Respuesta:

    Primeramente tiene un protocolo, el cuál es el método por el cuál se van a intercambiar los datos. Posteriormente sigue el dominio, el cuál representa el servidor web al cuál se le solicitaron los datos. En caso de que se quiera acceder acceder a un puerto en específico, se tendría que poner 2 puntos, y posteriormente el número del puerto a acceder, en caso de que no se especifique el puerto de entrada, se usaría el puerto 443 para HTTPS y 80 para HTTP. Después está la ruta, que antes representaba la dirección de un archivo, sin embargo, hoy en día es una abstracción que manejan los servidores. Casi para finalizar se encuentran los parámetros, los cuales son una lista de variables con valores; se separan con un ampersand(&). El último elemento de una URL son las anclas, éstas representan un punto o marcador en la página, éste es usado para que cuando se cargue la página, el navegador se desplace hasta dicho punto.

  3. ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?
    Respuesta:

    Cada método realiza una función diferente, éstos son sus propósitos:

    • GET:Solicita un recurso en específico al servidor, es decir, únicamente obtienen datos.
    • HEAD:Muy parecido al método GET, obtiene datos, sin embargo, obtiene únicamente la cabeza de la respuesta, sin el cuerpo.
    • POST:Su función es enviar datos al servidor, donde muchas veces efectúa algún cambio en el servidor.
    • PUT:Es usado para añadir, o en caso de ya existir un elemento destino, reemplazarlo.
    • PATCH:Éste método es usado para modificar parcialmente un recurso.
    • DELETE:Como indica su nombre, es usado para borrar un elemento.
  4. ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?
    Respuesta:

    El método POST, ya que tiene que el método tiene que ser un método seguro, porque si se usara un método como el GET, datos sensibles como la contraseña se podrían ver comprometidos, ya que en el método GET, los datos saldrían en la barra de direcciones.

  5. ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?
    Respuesta:

    Utilizan el método GET, ya que están solicitándole al servidor donde se aloja dicha página, los datos de la misma.

  6. Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?
    Respuesta:

    Significa un "ok", e otras palabras, que no ha sucedido ningún error y que la solicitud fue exitosa.

  7. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?
    Respuesta:

    No necesariamente, en algunos casos si. Primeramente se tiene que definir que el error 404 significa que no se ha encontrado el recurso solicitado. En el caso de que el usuario le haya presionado a un botón o enlace dentro de la aplicación web, y eso lo llevó al error, entonces SI se tendría que corregir, ya que la misma aplicación estaría llevándolo a un vínculo roto el cuál no existe. En caso de que el sitio al que intentó acceder el usuario lo colocó él mismo en le URL, entonces no se tendría que corregir nada, ya que el mismo usuario fue el que intentó acceder a algo que no existe.

  8. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?
    Respuesta:

    Si, el error 500 significa que tuvo algún tipo de error interno en el servidor, por lo que el desarrollador tendría que configurar el servidor de manera correcta para poder prevenir este tipo de errores.

    Respuesta:

    Significa que es un elemento el cuál no es recomendado su uso debido a que dicho elemento podría llegar a ser removido y llegue a perder compatibilidad con los navegadores.

    Algunos de éstos elementos son:

    • font: especificaba la fuente a usar en la página, junto con su color y tamaño.
    • center: centraba un texto.

  10. ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?
    Respuesta:

    En primera instancia, hay nuevas etiquetas que tienen distintas funciones cada una, sin embargo, hay más diferencias, tales como:

    • HTML4 no tiene soporte para audio y video mientras HTML5 si.
    • HTML4 era compatible con vectores como Flash y VML, mientras que HTML5 es compatible con SVG, canvas y otros elementos vectoriales.
    • HTML5 permite crear diversas figuras, como rectángulos, triángulos o círculos, mientras que HTML4 no.
    • La ejecución de JavaScript en HTML5 se realiza en hilos separados, mientras que en HTML4 se ejecutan en el mismo.
    • HTML4 usaba únicamente las cookies y el caché del navegador para almacenar información de forma temporal; HTML5 usa la misma base de datos de la página, el almacenamiento local y el caché.
    • HTML5 soporta los caracteres de UTF-8, mientras que HTML4 usa la codificación de caracteres de ASCII.

    tfoot Contiene los elementos de la última fila de la tabla, en otras palabras, el pie de la tabla.

  16. Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?
    Respuesta:

    En caso de ser posible, no usarlo, ya que puede causar que ciertos estilos no se apliquen debido a la jerarquía de prioridades. En dado caso, mejor revisar qué está afectando el estilo del elemento para modificarlo y evitar así el uso del !important.

  17. Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?
    Respuesta:

    Se tiene que escoger con cuidado ya que podría causar problemas de accesibilidad. En caso de que un usuario use algún lector de pantalla, el fondo de pantalla no podrá ser leído por éste último, por lo que, si la información que contenga el fondo de pantalla es importante, lo mejor sería que se describa de forma semántica.

  18. Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?
    Respuesta:

    Depende de lo que se quiera hacer, ya que hay casos donde conviene que una medida sea absoluta, y hay casos donde conviene que sea relativa. Para poner un ejemplo, el tamaño de una imagen podría ponerse de un tamaño relativo, es decir, que tenga un tamaño distinto dependiendo de la pantalla donde se vea, mientras que el texto conviene que sea de un tamaño absoluto.

  19. ¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?
    Respuesta:

    Debido a que una versión no minimizada cuenta con saltos de línea y espacios, los cuales no son necesarios al momento de renderizar la hoja de estilos, y solo hacen que pese más el archivo. Es decir, una versión minimizada contiene la información necesaria para la renderización, sin datos innecesarios que hagan más pesado el archivo.

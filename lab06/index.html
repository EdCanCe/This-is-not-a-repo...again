~/Preguntas

  1. ¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?

    Ya que permiten que si hay un error en el formato de los inputs que hizo el usuario comparados con los requeridos para subirlos a la base de datos, se le pueda alertar al usuario para que pueda corregirlos de inmediato. De igual manera, se podría implementar otra validación dentro del lado del servidor, sin embargo, ya que se tendrían que enviar los datos, y luego recibir una respuesta, podría añadir un retraso para que el usuario corriga los datos.

  2. ¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?

    Se puede saltar al incorporar validaciones en de otras formas. Por ejemplo, realizar la validación con un patrón de una expresión regular dentro del archivo HTML, usar una API, o realizar la validación en el lado del servidor.

  3. Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?

    Debido a que perimte que se pueda dar una respuesta inmediata a un usuario en caso de que alguno de los inputs no cumpla con las especificaciones que pide el formulario.
